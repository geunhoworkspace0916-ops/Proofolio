### [0:50 – 1:30] Solution + Why Blockchain (슬라이드 3) ⭐ 적절성 20점

> "Proofolio의 핵심은 한 문장입니다.
>
> **'PDF가 아니라 발급기관의 온체인 서명을 믿는다.'**
>
> 왜 블록체인이어야 하느냐, 세 가지입니다.
> 첫째, **운영자조차 못 바꿉니다** — 정책이 아니라 수학적으로요.
> 둘째, **누구나 Etherscan에서 독립 검증**할 수 있습니다.
> 셋째, **파일 한 글자만 바뀌어도 해시 불일치**로 즉시 잡아냅니다.
>
> DB는 결국 운영자 신뢰 게임으로 돌아가지만, 블록체인은 그 게임 자체를 끊어냅니다."

### [3:00 – 3:55] Architecture & 한계 (슬라이드 5) ⭐ 완성도 20점

> "기술적으로 단순한 데모가 아닙니다.
>
> ERC-721 표준에 **소울바운드** 패턴 — 증명서가 거래되면 안 되니까 양도를 막았고요. **Ownable2Step**과 **재진입 방지** 같은 스마트컨트랙트 보안 패턴을 적용했습니다.
> **Hardhat 단위 테스트 17개**로 정상·엣지·보안 케이스를 다 검증했고, **Etherscan에 verified** 되어 있어서 누구나 소스코드를 직접 확인할 수 있습니다.
>
> **한계도 솔직히 말씀드립니다** — 블록체인은 '발급 사실'만 보장하고, '그 기관이 좋은 기관이냐'는 보장 못 합니다. 그 영역은 W3C Verifiable Credentials 표준에서도 별도 신뢰 앵커로 해결합니다. 저는 관리자 레지스트리로 그 역할을 단순화했고요.
> 어떤 부분이 블록체인의 강점이고 어떤 부분은 한계인지 **명확히 알고** 만들었습니다."

## 🎥 발급 영상 녹화 가이드 (한 번만 하면 끝)

### 무엇을 녹화하나
**15~20초** 짜리 짧은 클립 — 발급기관이 보유자에게 증명서를 발급하는 흐름.

### 본인 노트북에서 (MetaMask 있는 곳)
1. 브라우저에서 `https://proofolio.pages.dev` 또는 로컬 `npm run dev` 띄우기
2. MetaMask 발급기관 계정 연결, Sepolia
3. **`/issue` 화면 준비** — 보유자 주소·종류·파일 미리 입력해 둠 (녹화 중 타이핑하지 말기)
4. **녹화 시작**: `Cmd + Shift + 5` → 브라우저 영역 선택 → **옵션에서 마이크 OFF, 마우스 클릭 표시 ON** → 녹화
5. 다음 순서로 천천히 (5초 간격):
   - 폼이 채워진 상태에서 잠깐 정지 (1~2초)
   - **[증명서 발급]** 클릭
   - MetaMask 팝업 → **확인** 클릭
   - 완료 화면 뜨면 `tokenId #N` 보이도록 정지 (3초)
6. 메뉴바 ⏹ → 자동으로 Desktop에 `.mov` 저장
7. 파일명을 알기 쉽게 변경: `발급_흐름.mov`

### 영상 편집 (선택, 30초 작업)
- macOS QuickTime에서 열기 → **Edit → Trim** 으로 앞뒤 빈 부분 자르기
- 18초 정도가 이상적

### Keynote에 임베드
1. 슬라이드 4 열기
2. `.mov` 파일을 슬라이드 위로 **드래그&드롭**
3. 우측 패널 **"동영상"** 탭 →
   - **자동 재생: 체크** (슬라이드 뜨자마자 재생)
   - **반복: 끄기**
   - **소리: 음소거**
4. Keynote 파일 저장(.key) — 영상이 패키지 안에 포함됨
5. **데모 PC에서 한 번 재생 테스트** 필수

### PowerPoint(.pptx)에 임베드도 동일
- 슬라이드 4 → **삽입 → 비디오 → 이 디바이스** → `.mov` 선택
- 우측 "비디오 재생" 옵션 → "자동 실행" 체크
